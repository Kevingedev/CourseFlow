# Historia de Usuario: Solicitud de participación en cursos

**ID:** `HU-003` | **Prioridad:** `Alta` | **Estimación:** `M`

## Descripción
**Como** usuario  
**Quiero** solicitar participación en un curso  
**Para** poder inscribirme en una formación disponible

## Criterios de Aceptación

### Escenario 1: Solicitud exitosa
- **Dado** que el usuario visualiza un curso disponible
- **Cuando** selecciona la option "Solicitar participación"
- **Entonces** el sistema registra la solicitud correctamente

### Escenario 2: Usuario no autenticado
- **Dado** que un usuario no ha iniciado sesión
- **Cuando** intenta solicitar participación en un curso
- **Entonces** el sistema solicita autenticación antes de continuar

### Escenario 3: Solicitud duplicada
- **Dado** que el usuario ya solicitó participación en un curso
- **Cuando** intenta volver a solicitar el mismo curso
- **Entonces** el sistema impide la acción y muestra un mensaje informativo

## Notas
- Las solicitudes serán almacenadas en JSON Server.
- Endpoint mock sugerido: `/requests`
- La validación de duplicados se realizará inicialmente en frontend.
