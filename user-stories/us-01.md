# Historia de Usuario: Registro de usuarios en la plataforma

**ID:** `HU-001` | **Prioridad:** `Alta` | **Estimación:** `M`

## Descripción
**Como** usuario  
**Quiero** registrarme en la plataforma  
**Para** poder acceder a las funcionalidades disponibles

## Criterios de Aceptación

### Escenario 1: Registro exitoso
- **Dado** que el usuario se encuentra en la pantalla de registro
- **Cuando** completa correctamente el formulario y envía los datos
- **Entonces** el sistema registra al usuario y muestra un mensaje de éxito

### Escenario 2: Validación de campos obligatorios
- **Dado** que el usuario se encuentra en la pantalla de registro
- **Cuando** intenta enviar el formulario con campos vacíos o inválidos
- **Entonces** el sistema muestra mensajes de validación correspondientes

### Escenario 3: Contraseñas no coinciden
- **Dado** que el usuario completa el formulario de registro
- **Cuando** la contraseña y la confirmación no coinciden
- **Entonces** el sistema impide el registro y muestra un mensaje de error

## Notas
- El frontend será desarrollado en Vue 3 (Composition API).
- Se utilizará JSON Server como mock API temporal.
- La persistencia de sesión se manejará mediante **cookies del navegador** para asegurar la compatibilidad con el backend definitivo.
- Se utilizará un **token predefinido** (`mock-jwt-token-xyz123`) para simular la autenticación exitosa en la fase de mock.
- Endpoint mock sugerido: `/users`

## Consideraciones de Escalabilidad
- **Transición a Backend Python:** El sistema debe estar diseñado para sustituir el mock por una API en Python que implementará autenticación mediante **JSON Web Token (JWT)**.
- **Seguridad de Cookies:** En esta fase se usarán cookies estándar, pero el código debe estar preparado para configurar atributos `HttpOnly` y `Secure` cuando se conecte al backend real.
- **Interceptores de API:** Se recomienda el uso de interceptores (ej. Axios) para adjuntar automáticamente el token de las cookies en las cabeceras de las peticiones futuras.
